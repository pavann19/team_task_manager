# Team Task Manager (Full-Stack)
A role-based task management system with structured validation, analytics, and workflow monitoring.

## 🚀 Live Demo
🔗 [Open Live Demo](https://team-task-manager-pj7e.onrender.com/)

## 📌 Overview
The Team Task Manager is a full-stack application designed to streamline team collaboration and project tracking. It provides a secure, role-based environment where administrators can manage projects and oversee workflows, while team members can efficiently track and update their assigned tasks. The system prioritizes data integrity and high-performance querying to deliver real-time metrics without over-engineering the architecture.

## ✨ Features
- **Authentication (JWT):** Secure token-based user authentication.
- **RBAC (Admin vs Member):** Strict separation of privileges at the API boundary.
- **Task Management:** Full CRUD capabilities for projects and tasks.
- **Priority & Due Date:** Built-in scheduling and task prioritization mechanics.
- **Analytics Dashboard:** Live metrics aggregating system-wide task statuses.
- **Overdue Tracking:** Automated detection and highlighting of missed deadlines.
- **Validation & Error Handling:** Comprehensive data validation yielding structured JSON responses.

## 🛠 Tech Stack
- **Backend:** FastAPI, Python 3.10+
- **Database:** PostgreSQL (Production) / SQLite (Local)
- **Frontend:** HTML, Vanilla JS, Tailwind CSS

## 🧠 Architecture
The application utilizes a **monolithic FastAPI architecture**. The backend serves the static frontend assets (HTML/JS/CSS) alongside the RESTful API routes. 

We chose to use Vanilla JS with Tailwind CSS instead of a heavy frontend framework like React. This approach maximizes architectural simplicity, drastically reduces build-time complexity, and inherently bypasses cross-origin (CORS) complications since the API and frontend share the exact same origin.

## 🔐 Role-Based Access Control
The system enforces strict Role-Based Access Control (RBAC) via FastAPI dependency injection:
- **Admin:** Full access to create projects, assign tasks, edit sensitive task metadata (priority, due dates), and view system-wide analytics.
- **Member:** Read-only access to system projects. Members can only interact with tasks explicitly assigned to them, and their permissions are restricted to updating the task's `status` (e.g., moving a task to 'Completed').

## 📊 Analytics & Data Processing
The analytics engine is designed for high performance and low memory overhead:
- **Server-Side Aggregation:** We utilize SQLAlchemy's native `func.count` and `group_by` aggregations directly at the database layer. 
- **Overdue Logic:** Overdue tasks are calculated via optimized SQL queries filtering for `due_date < now` and `status != 'Completed'`.
- **Efficiency:** Processing these metrics on the backend prevents transferring massive datasets over the network just for the frontend to count array lengths, ensuring the dashboard scales cleanly as the database grows.

## 🧩 Data Validation & Quality
Data integrity is maintained through a combination of database constraints and application-level schemas:
- **Pydantic Validation:** All incoming requests are strictly validated against Pydantic v2 schemas before reaching the business logic.
- **Structured JSON Responses:** The API exclusively returns a standardized `APIResponse` envelope (`success`, `message`, `data`) for predictable frontend parsing.
- **Error Handling:** Global exception handlers trap unhandled errors and HTTP exceptions, wrapping them in the standard JSON envelope to prevent internal server details from leaking to the client.

## ⚖️ Design Decisions
- **Single App Architecture:** Serving the static frontend directly from FastAPI minimizes deployment complexity and infrastructural overhead.
- **Server-Side Analytics:** Offloading metric calculations to the SQL engine drastically reduces application memory usage and network latency.
- **Minimal UI:** Relying on Tailwind utilities and Vanilla JS allows for rapid feature delivery without the overhead of maintaining a React component lifecycle.
- **Strong RBAC:** Enforcing permissions at the router dependency level ensures business logic remains secure by default, preventing vertical privilege escalation.

## 🔮 Future Improvements
While the core architecture is production-ready, the roadmap includes:
- **Pagination:** Implementing offset/limit query parameters to handle massive task volumes.
- **Rate Limiting:** Protecting authentication endpoints against brute-force attacks.
- **DB Migrations:** Integrating Alembic for seamless schema evolutions without data loss.
- **Advanced Analytics:** Expanding the dashboard to include time-to-completion metrics and user-specific performance charts.

## ▶️ How to Run Locally

1. **Clone the repository and install dependencies:**
```bash
pip install -r requirements.txt
```

2. **Run the local development server:**
```bash
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

3. **Access the application:**
Open your browser and navigate to `http://localhost:8000`.
